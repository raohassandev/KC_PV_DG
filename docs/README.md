(Slave ID 1), Port A (Slave ID 2)\n\nBenefit: Simpler Phase 1, lower hardware cost\nFallback: Dual ports if CRC errors >1% (noise isolation)\n```\n\n### Modbus Polling Frequency\n```\nT=0ms:    Poll Slave 1 (meter) → 80ms\nT=100ms:  Poll Slave 2 (inverter) → 100ms\nT=200ms:  Snapshot ready → Publish\nT=1000ms: Repeat\n\nResult: 1/sec per device, 2/sec snapshot (or 5/sec if low latency)\n```\n\n### Temperature Data\n```\nOLD: Physical sensors (DS18B20, 1-Wire)\nNEW: Modbus from inverter (Huawei exposes string temps)\nBenefit: No extra hardware, simpler maintenance\n```\n\n### Supported Devices (MVP)\n```\n✅ Rozwell EM500 (grid meter)      - Implemented\n🔨 Huawei Sun2000 (inverter)       - Phase 1 Week 3-4\n🔨 Carlo Gavazzi WM15 (meter)      - Phase 1 Week 2-3\n❌ Battery support                 - Out of scope\n```\n\n### Mobile Scalability\n```\n<DeviceTelemetry fieldMappings={getFieldsForBrand(brand)} />\n→ Works for ANY brand (generic component)\n\nRole-based access:\n- Owner: full permissions\n- Installer: view + configure\n- Support: view + logs\n```\n\n---\n\n## 📊 Document Statistics\n\n| Document | Sections | Lines | Focus
